# Project Structure and Data Flow

This README maps directories and key files so new contributors can navigate the repository quickly.

```
PLC_effect - Copy/
├── docs/                     # Detailed READMEs (this file, diffusion, Arrhenius, DSA, etc.)
├── inputs/
│   ├── bulk/                 # LAMMPS inputs/data for bulk simulations
│   └── interface/            # Interface diffusion input scripts and data files
├── outputs/
│   ├── bulk/                 # Logs, MSD, trajectories for bulk systems
│   ├── interface/            # Historical runs (pre-1k system)
│   ├── interface_1k/         # Current interface MSDs, logs, trajectories
│   └── analysis/             # Postprocessed results (diffusivity tables, plots, DSA outputs)
├── potentials/               # MEAM potential files and instructions
├── scripts/                  # Python utilities for setup, analysis, plotting
├── START_HERE.md             # Quick orientation (kept in root)
├── README.md / README_UPDATED.md
└── requirements.txt          # Python dependencies (SciPy, pandas, matplotlib)
```

---

## Directory Details

### `docs/`
- `README_OVERVIEW.md` – project context and milestones.
- `README_DIFFUSION_MD.md` – how to run interface diffusion MD.
- `README_INTERFACE_MODEL.md` – geometry and potential mapping.
- `README_ARRHENIUS.md` – Arrhenius fitting workflow.
- `README_MSD_QUALITY.md` – QA procedures for MSD data.
- `README_DSA_PLC.md` – PLC/DSA theory and predictions.
- `README_POTENTIALS.md` – MEAM potential notes.
- `README_PROJECT_STRUCTURE.md` – (this file).

### `inputs/`
- `interface/in.interface_diffusion` – main LAMMPS script controlling thermostat, output frequency, etc.
- `interface/interface_system_small.data` – default 1 k-atom bilayer (generated by `create_interface_system.py`).
- `interface/test_meam_minimize.in` – quick potential load sanity check.
- `bulk/` – retained for comparison or future bulk studies (not the focus of PLC analysis).

### `outputs/`
- `interface_1k/` contains the latest MSD data (`msd_interface_T*.dat`), profiles, and logs.
- `analysis/` holds CSVs and plots produced by Python scripts:
  - `interface_diffusivity_1k.csv` – raw diffusivity table.
  - `arrhenius_interface_*` – fit results and extrapolations.
  - `dsa_rho...csv`, `tau_comparison.png` – DSA/PLC analysis.
  - `length_scales_*.csv/txt` – documentation of L_c and L_t values.

### `scripts/`
- **Setup**: `create_interface_system.py`, `create_bulk_system.py`.
- **Analysis**:
  - `analyze_interface_msd.py`, `check_msd_quality.py`, `plot_interface_msd.py`.
  - `fit_arrhenius.py` – log-space Arrhenius regression and extrapolation.
  - `analyze_dsa.py` – τ_diff vs τ_wait comparison, parameter sweeps.
- **Reporting**: `generate_report.py`, plotting helpers, etc.

### `potentials/`
- `MgAlZn.library.meam`, `MgAlZn.parameter.meam` – Dickel et al. MEAM potential.
- Documentation: `README_POTENTIALS.md`, `POTENTIAL_SETUP.md`.

## Data Flow Summary

1. **System generation** – `create_interface_system.py` → `inputs/interface/interface_system_small.data`.
2. **MD runs** – `in.interface_diffusion` → `outputs/interface_1k/msd_interface_T*.dat`.
3. **MSD QA** – `check_msd_quality.py` / `plot_interface_msd.py`.
4. **Diffusivity table** – `analyze_interface_msd.py` → `outputs/analysis/interface_diffusivity_1k.csv`.
5. **Arrhenius fit** – `fit_arrhenius.py` → parameters & extrapolated diffusivity.
6. **DSA analysis** – `analyze_dsa.py` → τ_diff/τ_wait plots and CSVs.

Refer back to the individual READMEs for deeper context at each stage.

---

## Tips for New Contributors

1. **Clone and set up environment** – install requirements (`pandas`, `numpy`, `scipy`, `matplotlib`).
2. **Read `README_OVERVIEW.md`** to understand project goals and current status.
3. **Reproduce one MD run** (e.g., 850 K) and verify MSD QA before branching into new tasks.
4. **Keep outputs organized** – per temperature folder or tag runs by date if running multiple variants.
5. **Document changes** – update relevant README in `docs/` when workflow or assumptions change.

This structure prioritizes transparency and reproducibility; adjust as workflows evolve and ensure documentation stays synchronized.
